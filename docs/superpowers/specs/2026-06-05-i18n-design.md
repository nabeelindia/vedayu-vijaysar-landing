# Multilingual Support (i18n) — Design Spec
**Date:** 2026-06-05
**Status:** Approved

---

## Overview

Add Hindi, Tamil, and Telugu translations to the Vedayu customer-facing website using `next-i18next`. A language switcher in the top navbar lets users change language at any time. Preference is saved to `localStorage` and a cookie for SSR. Browser language is auto-detected on first visit. Blog posts and the admin panel are excluded from v1.

---

## Architecture

**Library:** `next-i18next` + `react-i18next`

**Locale files:**
```
public/locales/
  en/common.json   ← source of truth (English)
  hi/common.json   ← Hindi (हिन्दी)
  ta/common.json   ← Tamil (தமிழ்)
  te/common.json   ← Telugu (తెలుగు)
```

**`next.config.js`** gains:
```js
i18n: { locales: ['en', 'hi', 'ta', 'te'], defaultLocale: 'en' }
```

**New config file:** `next-i18next.config.js` at project root:
```js
module.exports = {
  i18n: { locales: ['en', 'hi', 'ta', 'te'], defaultLocale: 'en' },
  localePath: './public/locales',
};
```

---

## Language Detection & Persistence

1. On first visit: read `navigator.language` (e.g. `hi-IN` → `hi`, `ta` → `ta`)
2. If a match exists among supported locales, apply it automatically
3. Save preference to `localStorage` key `vedayu_lang`
4. Set `NEXT_LOCALE` cookie so SSR renders the correct locale on subsequent visits
5. `_app.js` reads `localStorage` on mount — if stored locale differs from active, calls `router.push(router.asPath, router.asPath, { locale })`

---

## Language Switcher UI

Located in the sticky top navbar (right side, alongside existing WhatsApp button).

- Displays current language as a pill: `🌐 EN ▾`
- Click opens a small dropdown showing all four options in their native script:
  - ✓ English
  - हिन्दी
  - தமிழ்
  - తెలుగు
- Selecting updates `localStorage`, sets `NEXT_LOCALE` cookie, triggers `router.push` with new locale — instant re-render, no full reload
- On mobile: dropdown anchors to the right edge of screen

## Pages Translated (v1)

| Page | File |
|------|------|
| Product + Checkout | `pages/index.js` |
| Order Confirmed | `pages/order-confirmed.js` |
| Order Tracking | `pages/track.js` |
| Contact | `pages/contact.js` |
| Footer | `components/SiteFooter.js` |

**Excluded from v1:** Admin panel, blog posts, static policy pages (privacy, terms, refund, shipping)

Each translated page/component gets:
```js
export const getServerSideProps = async ({ locale }) => ({
  props: { ...(await serverSideTranslations(locale, ['common'])) },
});
```

And uses:
```js
const { t } = useTranslation('common');
// <label>{t('checkout.name_label')}</label>
```

## Translation Quality Standard

All translations are handcrafted (not machine-generated) by a professional polyglot with expertise in Hindi, Tamil, Telugu, and English:

- **Hindi** — सरल, बोलचाल की हिन्दी। Conversational register. Uses common loanwords where natural (e.g. "ऑर्डर" for order, not "आदेश"). Correct gender agreement throughout.
- **Tamil** — நேர்மையான, எளிய தமிழ். Spoken Tamil register, not literary. Accessible to all age groups including elders.
- **Telugu** — సులభమైన, మాట్లాడే తెలుగు। Everyday spoken Telugu. Avoids overly formal or bureaucratic constructions.
- **English** — Simple, warm, direct. No jargon.

Interpolation placeholders (`{{variable}}`) are preserved exactly in all translations.

## Out of Scope (v1)

- Blog post translations (MDX files)
- Admin panel
- Static policy pages (privacy, terms, refund, shipping)
- RTL support (not needed — all three languages are LTR)
- Locale-specific number/currency formatting (₹ numerals universal)
- URL-based locale routing (e.g. `/hi/`) — locale stored in cookie only
